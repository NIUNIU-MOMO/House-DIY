# 标注页「查看确认 + 全屏编辑」设计

> **日期：** 2026-05-26  
> **状态：** 已确认  
> **关联需求：** [11-流程重构需求.md](../11-流程重构需求.md) §4.3

## 1. 目标

将步骤 3「标注」主页面收敛为**查看、保存、质检确认**；几何与房间编辑仅在**全屏手动标注**完成。比例尺标定从主页面左侧迁入全屏右侧中间区块。

## 2. 决策摘要

| 维度 | 决策 |
|------|------|
| 主页面画布 | 只读（B）：可选中高亮，不可拖顶点/边线/右键增房 |
| 主页面左侧 | 保留「选择」「显示」「房间列表」；移除比例尺与占位工具 |
| 主页面顶栏 | 不变：保存、状态、手动标注 |
| 主页面右侧 | 上：质检（无问题时显示「质检通过」）；下：确认户型 |
| 全屏右侧 | 三块：房间标注 / 比例尺 / 选中房间信息 |
| 实现方式 | `FloorPlanEditor` 主/全屏模板分支 + `FloorPlanSvg` 只读仍可选中 |

## 3. 信息架构

```text
主标注页
├── 左：选择 | 显示 | 房间列表
├── 中：只读画布 + 顶栏 [保存][状态][手动标注]
└── 右：质检 → 确认户型（贴底）

全屏手动标注
├── 顶：原图底图 | 保存 | 取消
├── 中：可编辑画布
└── 右：房间标注 | 比例尺 | 选中房间
```

## 4. 行为说明

### 4.1 主页面

- `FloorPlanSvg`：`editable=false`；扩展 `onRoomClick` / `onSvgClick` 在只读下仍触发 `roomSelect` / `backgroundClick`（大图预览）。
- 不展示顶栏质检横幅；质检迁入右侧 `data-testid="inspector-validation"`。
- 无问题时标题为「户型质检 · 通过」，中性/绿色样式。
- 不展示右侧手动标注面板、选中房间编辑、重画/删除。

### 4.2 全屏

- 保留矩形/描边、右键菜单、顶点/边线编辑；进入全屏默认矩形放置（既有逻辑）。
- 比例尺：右侧中间块，含状态文案、进入标定、两点+距离+应用。
- 「确认户型」仅主页面右侧，全屏不提供。

### 4.3 API 与数据

- `save` / `confirmFloorplan` / `validation` 语义不变。

## 5. 需求文档增量（FR-A18）

- **FR-A18a**：主标注页为查看确认态，编辑仅在全屏手动标注。
- **FR-A18b**：质检展示于主页面右侧；通过时显示「质检通过」。
- **FR-A18c**：比例尺标定入口位于全屏手动标注右侧中间区块。
